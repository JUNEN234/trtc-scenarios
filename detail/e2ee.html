Overview

End-to-End Encryption (E2EE) adds a layer of client-side encryption on top of TRTC calls and IM messages so that no intermediary — including Tencent's servers — can read or listen to the content. Designed for regulated industries: banking, healthcare, legal, and government.

Why Tencent RTC? TRTC already encrypts all streams with DTLS-SRTP, but E2EE adds an additional client-side encryption layer using customer-managed keys. TRTC's custom audio/video processing hooks allow encrypting before the media hits the server.

Key features

  • Client-side key exchange — ECDH key agreement between call participants, no keys touch the server.
  • Encrypted voice & video — AES-256-GCM encryption applied before TRTC transport.
  • Encrypted IM messages — client-side encrypt/decrypt with per-conversation keys.
  • Key verification — safety number display for manual verification (like Signal).
  • Forward secrecy — new ephemeral keys per session.
  • Compliance dashboard — admin panel showing encryption status, key rotation logs, audit trail.
