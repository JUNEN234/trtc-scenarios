Overview

Fan × Virtual Idol 1v1 Call is a mobile app where fans book real-time private video calls with AI-powered virtual idols. Each idol has a distinct personality (driven by an LLM persona engine), a cloned voice (ElevenLabs or Tencent Cloud TTS), and an animated Live2D / 3D avatar. Fans can also watch live broadcasts, send animated gifts, join fan clubs, and purchase coins.

Why Tencent RTC? The idol runs as a server-side TRTC participant — the backend joins the room as a bot and pushes synthesized TTS audio directly into the stream via trtc-server-sdk. Combined with streaming ASR + streaming LLM + streaming TTS, you keep end-to-end turn latency under 2.5 seconds while the fan's device stays light (just renders the avatar).

Key features

  • 1v1 private video calls with AI idols — LLM persona + cloned voice + Live2D/3D avatar, end-to-end latency ≤ 2.5s.
  • Live2D / 3D avatar expressions & lip-sync — driven by emotion tag + TTS audio RMS via TRTC IM data channel.
  • Persistent memory — Pinecone vector store, namespaced per fan × idol, idol remembers past conversations.
  • Real-time coin burn & queue — per-minute deduction; Redis sorted-set queue when idol is busy.
  • Virtual gifting during call — tier-based gifts inject in-character AI reactions via LLM pipeline.
  • Live broadcasts — idols stream to mass audiences with comment + gift + fan club tiers.
  • Content safety — every LLM output passes Tencent Content Safety before TTS.

Architecture

The AI call pipeline is the heart of the product. Here's the end-to-end flow during a voice turn:

pipeline 
Fan mic (PCM 20ms)
    ↓ WebSocket
Tencent ASR stream  →  partial transcript
    ↓  (silence > 800ms OR sentence end)
Embed → Pinecone top-5 memories
    ↓
LLM (streaming, JSON: content / emotion / thinkingNote)
    ↓
  ├─ emotion + lip-sync RMS  →  TRTC IM data channel  →  fan client avatar
  └─ content text  →  ElevenLabs/Tencent TTS  →  PCM stream
                                                      ↓  trtc-server-sdk
                                                TRTC room audio
                                                      ↓
                                             Fan hears idol's voice

Target end-to-end latency: ≤ 2.5 seconds. The idol bot runs as a separate worker process (BullMQ or Redis pub/sub), not in the API server.

TRTC custom audio injection (server-side idol bot)

Node.js 
import TRTCCloud from 'trtc-server-sdk';

const trtc = new TRTCCloud({ sdkAppId, secretKey });

// Bot joins the room as anchor
await trtc.enterRoom({
  roomId,
  userId: `idol_bot_${idolId}`,
  role: 'anchor',
});

// After TTS generates PCM audio buffer:
trtc.sendCustomAudioData({
  data: pcmBuffer,      // 16-bit PCM, 16kHz, mono
  sampleRate: 16000,
  channel: 1,
  timestamp: Date.now(),
});

Backend APIs

Call lifecycle:

  • POST /api/calls/start — coin balance check; provision TRTC room or queue entry; return credentials + queue position
  • POST /api/calls/:sessionId/end — final coin deduction, DismissRoom via TRTC REST, trigger memory extraction
  • GET /api/calls/history — paginated history
  • WebSocket /ws/call/:sessionId/audio — the full streaming pipeline: ASR → LLM → TTS → TRTC

Other routes:

  • Idols: catalog, profile, availability (queue length)
  • Gifts: deduct coins, inject gift event into WebSocket pipeline → idol AI reacts in character
  • Coins: Stripe PaymentIntent + webhook, Apple/Google IAP verify
  • Memory: GET /api/memory/:idolId, async extraction job post-call
  • Queue: Redis sorted set ZADD queue:{idolId}; ZPOPMIN + FCM push on slot free

Critical implementation notes

  1. UserSig is server-side only — never ship TRTC_SECRET_KEY to the client. Generate with tls-sig-api-v2.
  2. TRTC roomId must be a positive int32 — use Math.floor(Date.now() / 1000) % 2000000000.
  3. Idol bot is server-side — fan's device never calls LLM or TTS directly. Backend joins the room as idol_bot_{idolId} with anchor role.
  4. Run the pipeline in a worker process — use BullMQ or Redis pub/sub. Don't block the API server with LLM/TTS calls.
  5. Lip-sync = RMS per 20ms — send as audioEnergy: number[] via TRTC IM data alongside the audio push. Client animates mouth with 50ms lookahead buffer.
  6. Coin deduction every 60s via setInterval in the WebSocket handler. On balance = 0, send call_end_warning, 30s grace, then POST /api/calls/:sessionId/end.
  7. Queue = Redis sorted set. ZADD queue:{idolId} on start; ZPOPMIN + FCM push when a slot frees.
  8. Memory privacy — never persist raw call audio (unless opt-in). Pinecone vectors namespaced {fanUserId}_{idolId}.
  9. Network degradation — quality ≥ 5 → lower TTS bitrate; quality = 6 → pause pipeline, play pre-recorded idle phrases, overlay "Unstable connection".
  10. Gift reactions inject into the LLM pipeline — publish via Redis pub/sub as a special system message [GIFT EVENT: fan sent {type}. React warmly in character.], bypassing ASR.
  11. Content safety — every LLM output through Tencent Cloud Content Safety before TTS. On flag, substitute a safe fallback phrase matching the current emotion.
  12. Room cleanup — always call TRTC REST DismissRoom on call end; leaked rooms bill.
