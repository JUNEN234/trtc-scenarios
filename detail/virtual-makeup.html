Overview

Virtual Try-On Makeup is a mobile-first AR beauty experience where shoppers can see makeup products on their face in real time, get AI-powered shade recommendations tailored to their skin tone, share their look with friends via live video, and purchase in one tap. It combines on-device face mesh tracking, Lab-color-space makeup blending, and Tencent TRTC custom video source injection.

Why Tencent RTC? TRTC's custom video source API lets you inject the AR-composited frame (face + makeup overlay) directly into the outgoing stream instead of the raw camera feed — so the friend on the other end of the live call sees your makeup look, not your bare face.

Key features

  • Real-time AR makeup rendering — 468-point MediaPipe face mesh drives lip, eye, foundation, blush, contour, highlight, and brow layers at 30+ fps.
  • AI shade recommendation — skin tone + undertone analysis via Google Cloud Vision, personalized looks from GPT-4o.
  • Live video sharing — broadcast your AR try-on session to a friend via Tencent TRTC; they see the makeup overlay in real time.
  • Occasion-based looks — tell the AI "date night" or "job interview" and it picks a complete coordinated look from the catalog.
  • Save & re-apply — screenshot looks, re-apply later, share, or buy all products in one tap.
  • One-tap checkout — Stripe-powered purchase directly from the try-on screen.

Architecture

The AR + TRTC pipeline is the most latency-sensitive path. Here's the flow for a live-shared session:

pipeline 
  Camera frame (30fps)
        ↓
  MediaPipe 468-pt face mesh
        ↓
  Skia / CustomPainter canvas — render makeup layers in Lab space
        ↓
  Composite frame (camera + makeup overlay)
        ↓  extract pixel buffer (RGBA)
  TRTC sendCustomVideoData()  →  remote friend sees the look

Target: ≥ 30 fps on mid-range devices. Degrade to 20/15 fps on older hardware or when networkQuality ≥ 5.

AR frame injection into TRTC (the critical piece)

The most technically demanding part: TRTC must transmit the AR-composited frame, not the raw camera. Enable custom capture first, then push Skia/CustomPainter output via sendCustomVideoData.

React Native 
// Enable custom capture before entering the room
TRTCCloud.sharedInstance().enableCustomVideoCapture(
  TRTCVideoStreamType.big, true
);

// After each Skia render, export and push:
const snapshot = await skiaRef.current?.makeImageSnapshot();
const pixelData = snapshot?.encodeToBytes(); // RGBA Uint8Array

TRTCCloud.sharedInstance().sendCustomVideoData({
  data: pixelData,
  width: frameWidth,
  height: frameHeight,
  timestamp: Date.now(),
  pixelFormat: TRTCVideoPixelFormat.RGBA32,
  bufferType: TRTCVideoBufferType.byteArray,
});
Flutter 
trtcCloud.enableCustomVideoCapture(TRTCVideoStreamType.big, true);

final recorder = ui.PictureRecorder();
final canvas = Canvas(recorder);
arPainter.paint(canvas, size);
final picture = recorder.endRecording();
final image = await picture.toImage(width, height);
final byteData = await image.toByteData(format: ui.ImageByteFormat.rawRgba);

trtcCloud.sendCustomVideoData(TRTCVideoFrame(
  pixelFormat: TRTCVideoPixelFormat.rgba,
  bufferType: TRTCVideoBufferType.byteBuffer,
  data: byteData!.buffer.asUint8List(),
  width: width,
  height: height,
  timestamp: DateTime.now().millisecondsSinceEpoch,
));

Backend APIs

Skin tone analysis — POST /api/skin/analyze:

  1. Accept cropped cheek-region JPEG (base64)
  2. Google Cloud Vision IMAGE_PROPERTIES → dominant colors
  3. Determine undertone from hue bias (warm / cool / neutral)
  4. Nearest-shade match in Lab color space against product DB
  5. GPT-4o picks 3 foundation + 2 lip + 2 blush shades given tone + undertone + catalog
  6. Save SkinAnalysis, update User.skinTone — do not persist the image

AI look recommendation — POST /api/ai/recommend: streaming SSE response with GPT-4o picking a complete look given occasion + skin tone + undertone.

TRTC routes:

  • POST /api/trtc/room — server-side userSig via tls-sig-api-v2, create LiveRoom with numeric roomId
  • GET /api/trtc/room/:roomId/join — validate + issue guest userSig
  • POST /api/trtc/room/:roomId/end — call TRTC REST DismissRoom, save TryOnSession

Other routes: products (filterable), saved looks (CRUD + COS upload), Stripe checkout + webhook.

Critical implementation notes

  1. UserSig is server-side only — never ship TRTC_SECRET_KEY to the client.
  2. Frame-injection rate — target 30 fps; drop to 20 fps on older devices; always off the UI thread.
  3. TRTC roomId must be a positive int32 — no UUIDs. Use Math.floor(Date.now() / 1000) % 1000000.
  4. Blend in Lab color space, not RGB — the difference between realistic makeup and a flat color sticker.
  5. Fallback mesh — if detected fps < 20, drop to 68-point mesh and render lip + blush only.
  6. Memory discipline — release Skia snapshots + pixel buffers immediately after pushing to TRTC; holding multiple frames OOMs 2 GB devices.
  7. Skin-tone privacy — process cheek-crop image in memory, never persist; store only hex + undertone.
  8. Network degradation — quality ≥ 5 → 15 fps; quality = 6 → audio-only.
  9. Room cleanup — always call TRTC DismissRoom on session end; leaked rooms bill.
  10. Deep-link invites — yourapp://live/:roomId; if app not installed, redirect to App Store / Play Store and store roomId in Redis for post-install join.
